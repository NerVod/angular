Les modules

Nous utilisons déjà un module dans notre application : le module principal AppModule, défini dans src/app/app.module.ts.

Le décorateur @NgModule

Un module Angular est une classe affublée du décorateur @NgModule et qui ne contient généralement aucun code.

import { NgModule } from '@angular/core';

@NgModule({
  declarations: [...], // Liste de directives. Les composants et les filtres (« pipes ») sont des types de directives). Le tableau `declarations` contient donc un liste de classes de directives, composants ou pipes « pipes ». Il s’agit donc d’éléments qui peuvent être utilisés dans le template sous forme de mots-clés (directives).
  imports: [...], // Liste de modules à importer dans ce module. Tout ce qui vient d’un module importé ici est disponible pour les déclarations de ce module.
  exports: [...], // Liste des composants, directives, et filtres (« pipes ») visibles pour les modules qui importent ce module.
  providers: [...], // Liste de services ou, plus généralement, de fournisseurs (« providers ») d’injection de dépendances visibles à la fois aux contenus de ce module et à ceux qui importent ce module.
  entryComponents: [...], // NOTE : cette propriété n’est plus utilisée depuis Angular 11. Liste de composants qui ne sont pas référencés dans des templates, tels que des composants créés dynamiquement en code.
  bootstrap: [...] // Liste de composants à démarrer quand ce module est démarré.
})
class MyModule {}

Toute application Angular démarre dans un module, dit « module principal » ou « module racine » (nommé conventionnellement AppModule), mais une application Angular sera généralement composée de nombreux modules.

Les modules permettent de regrouper des services et composant par fonctionnalité, afin d’organiser le projet, mais aussi afin de limiter le code à charger initialement lorsqu’on utilise le lazy-loading.

Chaque nouvelle fonctionnalité peut être créée dans un module dédié. On peut aussi créer des modules partagés dont les fonctionnalités (composants, filtres, services…) peuvent être utilisées dans les autres modules.

Il est courant par exemple de créer un module par page de l’application, ce module regroupant tous les composants et services propres à cette page, et qui importe les modules dont elle a besoin.

Modules tiers

Les modules permettent non seulement d’organiser une application, ils facilitent aussi l’ajout de fonctionnalités issues de bibliothèques tierces.

Angular propose des bibliothèques de code qui sont des NgModules, comme FormsModule, HttpClientModule, et RouterModule.

De nombreuses bibliothèques tierce-partie sont disponibles sous forme de NgModules, comme par exemple Material Design, Ionic, et AngularFire2.

Exercice f1 : modularisation

Partie 1

À l’aide du CLI, créez un nouveau module nommé « exercices" à la racine de l’application.

Dans ce module, déclarez tous les composants des exercices précédemment créés et exportez-les.

Importez ce module dans votre module principal.

N’oubliez pas de supprimer les exercices des déclarations du module principal (un composant ne peut être déclaré qu’une seule fois par application !)
